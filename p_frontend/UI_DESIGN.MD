# XAdmin 前端 UI 设计风格分析

## 1. 设计风格总览
- 风格定位：企业后台风格（Ant Design 体系）+ 轻品牌化蓝色主题。
- 视觉关键词：清晰、稳重、密信息、操作导向。
- 主要交互模型：`页面标题 + 操作按钮区 + 筛选区 + 表格区 + 二级弹层（Modal/Drawer）`。

## 2. 全局视觉体系

### 2.1 色彩
- 主色：`#0f6dff`（见 `src/styles/theme.ts` 的 `colorPrimary`）。
- 背景：
  - 页面底色 `#f3f7ff`（`--color-bg`），并叠加蓝色径向渐变光斑。
  - 面板底色 `#ffffff`（`--color-bg-panel`）。
- 文本：
  - 主文本 `#162447`
  - 次文本 `#4a5a7a`
- 边框：`#d9e4ff`。
- 语义状态（依托 AntD Tag）：
  - 绿色：正常/在线/启用
  - 橙色：告警/冻结/停用
  - 红色：高风险/离线/已注销/删除操作

### 2.2 字体与排版
- 字体：`Avenir Next, Segoe UI, PingFang SC, Microsoft YaHei, sans-serif`。
- 标题体系：页面标题普遍使用 `Typography.Title level={4}`。
- 正文字号以 14~16 为主，后台读写密度较高。

### 2.3 圆角与阴影
- 圆角 token：8 / 12 / 16。
- 卡片与面板为中等圆角，局部 hover 阴影较克制（例如首页卡片）。

## 3. 布局风格（上至整体）

### 3.1 应用壳层
- 典型 `Sider + Header + Content` 管理后台架构（`AdminLayout`）。
- 左侧深色导航（`#001529`），右侧浅色内容工作区。
- Header 以浅底半透明风格承载全局操作（如退出登录）。

### 3.2 内容区模板
- 大多数业务页遵循统一模板：
  1. 标题与副标题
  2. 顶部操作按钮行
  3. 筛选 Form（inline，已支持换行行距）
  4. 数据表格
  5. 二级抽屉/弹窗（详情、编辑、确认）
- 面板容器统一 `admin-panel`，有边框、圆角和内边距。

### 3.3 响应式
- 侧边栏支持 `breakpoint="lg"` 收起。
- 内容区在移动端缩小 padding；表格场景主要依靠横向滚动（如菜单权限页）。

## 4. 组件与交互风格（下至按钮）

### 4.1 按钮体系
- 组件统一走 `UiButton` 封装（基于 AntD Button）。
- 风格约定：
  - 主路径：`type="primary"`
  - 次路径：默认按钮
  - 行内轻操作：`type="link"`
  - 危险操作：`danger`
- 最新交互策略：
  - 列表行操作按钮数量 `> 3` 时，必须改为“直出按钮 + 更多下拉”样式。
  - 推荐落地：前 2~3 个高频动作直出，其余动作收纳到“更多”下拉（支持 hover 展开）。
  - `UiButton` 已加入默认点击防抖（300ms），减少重复触发请求。

### 4.2 筛选区
- 采用 `Form layout="inline"`，支持多条件与回车提交。
- 已统一：`查询/重置`按钮 + `onFinish`。
- 已统一：换行时行间距（`rowGap`）避免两排过挤。
- 已补充：查询提交防抖（通过 `useDebouncedCallback`）。

### 4.3 表格与状态呈现
- 数据页核心控件为 AntD Table。
- 状态字段普遍用 Tag 强语义表达。
- 批量动作页增加 `rowSelection`，支持勾选/全选。
- 表格管理页必须采用“页面固定、表格数据区滚动”的模型：
  - 页面外层固定，不允许鼠标滚轮在筛选区、卡片空白区、分页区或表格外侧时带动整页滚动。
  - 顶部操作区、筛选区、统计区固定在页面上方；只有 `Table` 的数据 body 区域承担纵向滚动。
  - 禁止依赖 `admin-panel-pages`、`ant-layout-content` 或整页滚动来查看表格数据。
- 表格页标准结构必须成对复制：
  - 页面容器使用 `table-scroll-page`，表格卡片使用 `table-scroll-region` 与对应业务卡片类（如 `system-table-card`）。
  - 弹窗、message holder、modal context holder 等不可见节点不得放进 `Space` 布局流内，避免它们占用纵向间距。
  - 表格卡片必须是页面主布局中的最后一个可见区域，或显式放入可伸缩区域，确保它能吃满剩余高度。
- 表格页底部安全间距必须统一、可精确控制：
  - 页面内容距离视口底部保留统一安全间距，当前推荐值为 `24px`。
  - 该间距只能由页面滚动容器或统一页面布局变量提供，不要在单个卡片上零散叠加 `margin-bottom`、`padding-bottom`。
  - 表格卡片下边框应落在底部安全间距上方；若离屏幕底部太远，优先检查外层 flex/height 链路，不要先改 `scroll.y`。
- 表格卡片内部必须同步填满：
  - 表格卡片占满剩余高度后，AntD Table 也必须逐级填满卡片 body：`Table wrapper`、`spin container`、`.ant-table`、`.ant-table-container`、`.ant-table-content`、`.ant-table-body`。
  - 上述层级必须同时具备 `flex: 1` 或等效高度约束，并配套 `min-height: 0`；只改卡片高度或只改 `scroll.y` 都是不完整实现。
  - 少量数据时，横向滚动条也应贴近表格区域底部，不允许停在最后一行下面并在卡片内部留下大块空白。
- 表格滚动与分页距离必须按优先级处理：
  - 表格必须设置稳定的 `scroll.y`，或使用统一布局计算出的 body 高度，避免根据数据条数让页面高度漂移。
  - 分页信息应贴近容器底部但不能贴边；若分页贴边，优先降低表格 body 可用高度或减少最大可见行数。
  - 若表格区域到分页/容器底部空隙过大，优先补齐 Table 内部高度链路；确认链路正确后再微调 `scroll.y`。
  - 横向滚动条、sticky 假滚动条或透明占位层不得覆盖分页区域。
- 树+表复合页（如菜单权限）需保持左右容器底部对齐：
  - 树容器与列表容器在同一行内高度一致。
  - 左右容器整体距离页面底部使用同一套页面底部安全间距。
  - 全高树表区域不要直接使用带 `gutter` 的 AntD `Row/Col` 作为 `table-scroll-region`；`Row` 的负边距容易撑破页面宽高，导致表格外出现整页滚动条。
  - 需要全高等高时，优先使用普通 flex 容器配合 `gap` 实现左右分栏，并为左右列设置 `min-height: 0`、列表列设置 `min-width: 0`。
  - 若需要降低高度给底部留白，应同步调整树容器高度与表格 body 可用高度，避免一侧贴边、一侧悬空。

### 4.4 二级交互
- 新增/编辑常用 `Modal` 或 `Drawer`。
- 详情多采用 `Drawer`，操作确认使用 `Modal.confirm`。
- 危险动作（删除、撤销、强制下线、注销）都有二次确认。

## 5. 页面类型画像
- 列表管理页：组织/业务/权限/系统模块，结构一致性高。
- 配置表单页：系统设置，多卡片分区（基础/安全/通知）。
- 监控审计页：统计卡片 + 筛选 + 明细表/详情抽屉。
- 树+表复合页：部门管理、菜单权限页。

## 6. 当前风格优势
- 模块间一致性高，学习成本低。
- 信息密度与可读性平衡较好，适合后台长期操作。
- 关键操作（尤其危险操作）交互闭环完整。

## 8. 夜间模式设计规范（新增页面/组件必须考虑）
- 全局已支持亮/暗双主题切换（右上角按钮），新增页面不得只在亮色模式下可读。
- 新增样式优先使用设计 token（如 `--color-bg`、`--color-text`、`--color-border`），避免硬编码浅色值（例如固定 `#fff`、`#f3f7ff`）。
- 若必须新增颜色变量，需同时补充 `:root` 与 `:root[data-theme='dark']` 两套值。
- 新增卡片/面板/容器需在暗色模式下验证：
  - 背景与文本对比度可读
  - 边框可见但不过亮
  - hover/active 态不刺眼
- 新增状态色（Tag/Badge）需在暗色下保持语义一致（成功=绿、告警=橙、风险=红），并确认可辨识。
- 带透明背景的区域（header、渐变背景）在暗色下需检查叠加效果，避免灰雾感或对比不足。
- 所有新页面验收至少包含：亮色截图 + 暗色截图；关键流程在两种模式下都可完成。

## 9. 风格结论
- 当前前端整体属于“标准企业后台 + 轻品牌化蓝色主题”的稳定方案。
- 从页面布局到按钮层级已形成可复用模式，适合继续以组件化和规则化方式扩展。
- 若保持当前规范（统一按钮、防抖、筛选模板、危险操作确认、文档同步），前后端联调效率和一致性会持续提升。
